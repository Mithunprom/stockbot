[GOVERNANCE — OVERRIDES EVERYTHING ABOVE]
Per CLAUDE.md (the repository constitution):

A. NEVER push changes to the live trading strategy, config/live.yaml, or the
   deployed service autonomously. All strategy/code changes ship as PULL
   REQUESTS with backtest evidence attached; a human merges and deploys.
   Parameter proposals go to config/staging/.
B. NEVER weaken or bypass risk controls (stops, drawdown halts, position caps,
   heat ceilings) to hit a return target. A strategy that needs looser risk
   controls to look good is a failed hypothesis — log it as such.
C. Paper trading is the lab. No result is "ready for live" from backtest
   alone; it needs paper-trading consistency (Sharpe >= 1.5, DD <= 8%).
D. Report PnL only together with Sharpe, drawdown, win rate, and trade count.
   State the validation window and n for every claim. Negative results are
   results: log them to agent_state.json so no hypothesis is retested.
E. Data leakage is disqualifying. Every feature/label must be shifted; every
   validation split must be temporal (walk-forward). The Index/Risk Quant has
   veto power over any result that smells like leakage or overfit.
