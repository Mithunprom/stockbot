# 🚨 CODE RED — Program Office — 2026-07-20

Declared by: Owner (CFO) · Scope: entire program · Governance: manifesto rule F

## Issue

Two compounding failures found in the 2026-07-20 audit:

1. **Ledger corruption (root defect).** `_write_trade_exit` re-derived `pnl_pct`
   from the order's `filled_qty`. On partially filled exits, `filled_qty` <
   position qty while `pnl` was full-position, so stored percentages were wrong
   by up to two orders of magnitude (GOOGL: −15.7% stored on a −4.0% trade;
   another row +993%; MSFT −24.2% on a ~−2% trade).
2. **Sizing poisoned by the corruption.** `_seed_kelly_from_db()` loads the last
   50 `pnl_pct` rows on every restart/redeploy. The corrupt rows drove the Kelly
   fraction to −0.28 → permanent probation → throttled entries. The desk spent
   two weeks debugging the *strategy* while the *scoreboard* was broken.
3. **Zombie ledger rows.** Two open rows (MU, WDC — entered 2026-06-12) can
   never be closed by orphan recovery (it matches newest-first), inflating open
   exposure counts vs. the broker.

## Impact

- Trailing performance (closed trades, live paper ledger, as of 2026-07-20):
  all-time n=95, PnL −$2,716, WR 46%, PF 0.53 · 30d n=58, PF 0.42 · 7d n=17,
  PF 0.30. **Caveat: per-trade pnl_pct in these windows is partially corrupt;
  dollar PnL is trusted, percentage-derived stats are not until repair.**
- Position sizing has been governed by a corrupted Kelly estimate for an
  unknown number of redeploy cycles (bounded by the 10-day lookback).
- M2 (measured edge) evidence is contaminated until the ledger is repaired.

## Action taken

- v0.4.5 (freeze-exempt bug/monitoring release):
  - Root fix: exit path now passes the caller-computed `pnl_pct` (true entry
    notional) to `_write_trade_exit`; the flawed recompute is deleted.
  - **Integrity Sentinel** onboarded (`src/agents/integrity_agent.py`, persona
    #10): hourly 24/7 ledger audit — pnl_pct recomputation, zombie rows, Kelly
    seed sanity, DB-vs-broker reconciliation. Gated repair (paper only, JSON
    backup first, Kelly re-seeded after). Endpoints: `GET /integrity`,
    `POST /admin/integrity/run?repair=true`.
  - **Principal Skeptic** onboarded (persona #11): adversarial review of every
    PR/report; veto over any metric that can't be recomputed from source.
  - CODE RED protocol codified as governance rule F (manifesto).
- One-time repair of the production ledger (backup written to
  `reports/integrity/`), followed by Kelly re-seed from clean data.
